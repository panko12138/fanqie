## 1 产品概述

### 1.1 产品定位

考研番茄钟是一款专为考研学子设计的桌面专注力管理工具。它基于经典的番茄工作法（Pomodoro Technique），结合考研复习的实际场景，帮助用户管理学习时间、跟踪学习进度、培养良好的学习习惯。

### 1.2 核心价值

- **时间管理**：通过番茄钟循环制造专注时段与休息时段的节奏感
- **任务管理**：将考研复习任务拆解为可执行的小目标，关联番茄数量
- **数据可视化**：统计每日/每周/每月的学习数据，直观看到进步
- **习惯养成**：连续打卡机制激励用户坚持学习

### 1.3 目标用户

| 属性 | 描述 |
|------|------|
| 用户类型 | 备考研究生（本科/专科均可） |
| 年龄范围 | 18 - 30 岁 |
| 技术背景 | 普通用户，无需编程基础 |
| 使用场景 | 图书馆、自习室、家中 |
| 核心诉求 | 简单易用、不分心、能看到进步 |

## 2 用户分析

### 2.1 用户痛点

1. 学习时容易分心，拿起手机就停不下来
2. 不知道自己每天真正学习了多少时间
3. 复习进度模糊，无法量化每个科目的学习时长
4. 缺乏持续学习的动力和反馈
5. 现有番茄钟 App 功能过于复杂或广告多

### 2.2 用户故事

小明是一名大三备考学生，每天在图书馆学习 10 小时以上，但经常觉得效率低下。他希望有一个工具能帮他：

- 每天上午用 4 个番茄完成高数复习
- 下午用 3 个番茄做英语阅读
- 晚上用 2 个番茄复习政治
- 看到每周的学习时间统计，感受到进步
- 连续打卡 30 天后获得成就感

## 3 功能需求

### 3.1 核心计时器

#### 3.1.1 番茄钟循环

实现经典的番茄工作法循环机制：

| 阶段 | 默认时长 | 说明 |
|------|---------|------|
| 专注时段 | 25 分钟 | 全神贯注学习，不可中断 |
| 短休息 | 5 分钟 | 每个番茄之后休息 |
| 长休息 | 15 分钟 | 每 4 个番茄后长休息 |

#### 3.1.2 计时器操作

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 开始 | Space | 开始/继续计时 |
| 暂停 | Space | 暂停当前计时 |
| 重置 | R | 重置当前阶段计时 |
| 跳过 | S | 跳过当前阶段，进入下一阶段 |
| 停止 | Esc | 完全停止计时并重置 |

#### 3.1.3 自定义时长设置

用户可在设置中自定义各阶段时长：

| 阶段 | 默认时长 | 可调整范围 | 说明 |
|------|---------|-----------|------|
| 专注时段 | 25 分钟 | 10-60 分钟 | 全神贯注学习，不可中断 |
| 短休息 | 5 分钟 | 3-15 分钟 | 每个番茄之后休息 |
| 长休息 | 15 分钟 | 10-30 分钟 | 每 N 个番茄后长休息（N 可自定义） |
| 长休息间隔 | 4 个 | 2-8 个 | 每完成多少个番茄后进行长休息 |

#### 3.1.4 专注模式

- **窗口置顶**：计时器窗口始终保持在最上层
- **白噪音**：提供多种白噪音选项（雨声、森林、咖啡馆等）帮助专注
- **通知屏蔽**：专注时段屏蔽系统通知（可选）
- **全屏模式**：隐藏其他应用，只显示计时器界面

#### 3.1.5 计时器界面要素

- **圆形进度条**：实时显示剩余时间占比
- **数字倒计时**：大字号显示 MM:SS 格式
- **状态指示**：当前阶段名称（专注/短休息/长休息）
- **番茄计数**：显示当前循环已完成的番茄数（如 2/4）
- **当前任务**：显示正在进行的任务名称

### 3.2 任务管理

#### 3.2.1 任务属性

| 属性 | 类型 | 说明 |
|------|------|------|
| 任务名称 | 文本 | 必填，最多 50 字 |
| 科目分类 | 枚举 | 政治 / 英语 / 数学 / 专业课 / 其他 |
| 预估番茄数 | 数字 | 预计需要几个番茄完成 |
| 实际番茄数 | 数字 | 自动累加，完成番茄时 +1 |
| 优先级 | 枚举 | 高 / 中 / 低 |
| 创建日期 | 日期 | 自动记录 |
| 完成状态 | 布尔 | 进行中 / 已完成 |
| 备注 | 文本 | 可选，详细任务描述 |

#### 3.2.2 任务操作

- **新建任务**：填写名称、选择科目、设置预估番茄数
- **编辑任务**：修改任务名称、科目、优先级
- **删除任务**：删除后不影响已记录的统计数据
- **完成任务**：手动标记完成，实际番茄数达到预估时自动提醒
- **选择当前任务**：点击任务后，下一个番茄自动关联此任务
- **任务拖拽排序**：可拖拽调整任务顺序
- **批量操作**：支持批量删除、批量完成任务
- **任务模板**：提供常用考研任务模板（如"数学一章复习"、"英语阅读一篇"等）
- **任务备注**：可为任务添加详细备注信息

### 3.3 统计分析

#### 3.3.1 数据维度

| 维度 | 指标 | 展示形式 |
|------|------|---------|
| 今日 | 完成番茄数、专注时长、完成任务数 | 数字卡片 |
| 本周 | 每日番茄数趋势 | 柱状图 |
| 本月 | 每日专注时长趋势 | 折线图 |
| 科目 | 各科目时间分布 | 饼状图 |
| 累计 | 总番茄数、总专注时长、连续打卡天数 | 数字卡片 |

#### 3.3.2 打卡机制

- 每天完成至少 1 个番茄即计为当天打卡成功
- 显示当前连续打卡天数和历史最长连续打卡天数
- 设置每日番茄目标数（默认 8 个），达成后显示特别提示
- 日历视图：直观展示每日打卡情况和番茄完成数
- 成就系统：完成特定目标后解锁成就徽章
- 分享功能：可分享每日/每周学习数据到社交媒体（可选）
- 数据导出：支持导出统计数据为图片或PDF格式

### 3.4 提醒系统

#### 3.4.1 提醒方式

- **音效提醒**：番茄结束时播放提示音（可选择不同音效）
- **系统通知**：窗口最小化时发送系统通知
- **弹窗提醒**：番茄结束时弹出提示对话框

#### 3.4.2 休息提示语

休息阶段随机显示健康提示语，例如：

- "站起来活动一下吧！"
- "闭眼休息 30 秒，让眼睛放松一下。"
- "喝杯水，保持水分充足。"
- "做做拉伸运动，缓解颈椎疲劳。"

### 3.5 考研倒计时

- 用户可设置考研日期
- 主界面顶部显示"距离考研还有 XX 天"
- 配合鼓励文案，如"坚持就是胜利！"

### 3.6 系统托盘

- 最小化到系统托盘，后台运行
- 托盘图标显示当前状态（计时中/空闲）
- 右键菜单：显示/隐藏主窗口、开始计时、退出
- 悬停提示：鼠标悬停显示剩余时间和当前状态

### 3.7 设置功能

#### 3.7.1 外观设置

- **主题切换**：支持浅色/深色/自动主题
- **字体大小**：可调整界面字体大小
- **界面语言**：支持中文/英文切换
- **透明度**：可调整窗口透明度（可选）

#### 3.7.2 通知设置

- **音效选择**：提供多种提醒音效（可自定义音效文件）
- **音效音量**：独立调节提醒音效音量
- **系统通知**：开启/关闭系统通知
- **弹窗提醒**：开启/关闭弹窗提醒

#### 3.7.3 数据管理

- **自动备份**：支持每日自动备份数据库
- **手动备份**：手动创建数据库备份
- **数据恢复**：从备份文件恢复数据
- **数据清理**：可删除指定日期之前的历史数据
- **数据导出**：导出所有数据为JSON格式
- **数据导入**：从JSON文件导入数据

#### 3.7.4 数据库配置

- **MySQL 连接设置**：配置主机、端口、用户名、密码、数据库名
- **连接测试**：测试数据库连接是否成功
- **初始化脚本**：首次使用时自动创建数据库和表

### 3.8 无障碍支持

- **键盘导航**：完整支持键盘操作所有功能
- **屏幕阅读器**：支持主流屏幕阅读器
- **高对比度**：提供高对比度主题
- **大字体模式**：超大字体显示模式

## 6 数据库设计

### 6.1 数据库配置

首次使用时需要配置 MySQL 连接信息：
- 主机：默认 localhost
- 端口：默认 3306
- 用户名：需要有创建数据库权限
- 密码：用户密码
- 数据库名：pomodoro_db

### 6.2 数据表结构

#### 6.2.1 tasks 表（任务表）

```sql
CREATE TABLE tasks (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(255) NOT NULL COMMENT '任务名称',
    subject VARCHAR(50) NOT NULL COMMENT '科目分类',
    estimated_pomodoros INT DEFAULT 1 COMMENT '预估番茄数',
    actual_pomodoros INT DEFAULT 0 COMMENT '实际番茄数',
    priority VARCHAR(20) DEFAULT 'medium' COMMENT '优先级',
    status VARCHAR(20) DEFAULT 'active' COMMENT '状态',
    notes TEXT COMMENT '任务备注',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    completed_at DATETIME COMMENT '完成时间',
    INDEX idx_status_created (status, created_at),
    INDEX idx_subject (subject)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='任务表';
```

#### 6.2.2 pomodoro_sessions 表（番茄记录表）

```sql
CREATE TABLE pomodoro_sessions (
    id INT PRIMARY KEY AUTO_INCREMENT,
    task_id INT COMMENT '关联任务ID',
    start_time DATETIME NOT NULL COMMENT '开始时间',
    end_time DATETIME NOT NULL COMMENT '结束时间',
    duration INT NOT NULL COMMENT '实际专注时长（秒）',
    completed BOOLEAN DEFAULT TRUE COMMENT '是否完成（vs 中断）',
    INDEX idx_task_id (task_id),
    INDEX idx_start_time (start_time),
    FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='番茄记录表';
```

#### 6.2.3 daily_stats 表（每日统计表）

```sql
CREATE TABLE daily_stats (
    date DATE PRIMARY KEY COMMENT '日期',
    total_pomodoros INT DEFAULT 0 COMMENT '当日完成番茄数',
    total_focus_time INT DEFAULT 0 COMMENT '当日总专注时长（秒）',
    tasks_completed INT DEFAULT 0 COMMENT '当日完成任务数',
    daily_goal INT DEFAULT 8 COMMENT '每日番茄目标',
    goal_achieved BOOLEAN DEFAULT FALSE COMMENT '是否达成目标',
    INDEX idx_date (date)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='每日统计表';
```

#### 6.2.4 settings 表（设置表）

```sql
CREATE TABLE settings (
    `key` VARCHAR(100) PRIMARY KEY COMMENT '设置项名称',
    `value` TEXT NOT NULL COMMENT '设置项值',
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='设置表';
```

#### 6.2.5 achievements 表（成就表）

```sql
CREATE TABLE achievements (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '成就名称',
    description TEXT NOT NULL COMMENT '成就描述',
    icon VARCHAR(255) NOT NULL COMMENT '成就图标',
    `condition` TEXT NOT NULL COMMENT '解锁条件',
    unlocked_at DATETIME COMMENT '解锁时间',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='成就表';
```

#### 6.2.6 task_templates 表（任务模板表）

```sql
CREATE TABLE task_templates (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(255) NOT NULL COMMENT '模板名称',
    subject VARCHAR(50) NOT NULL COMMENT '科目分类',
    estimated_pomodoros INT DEFAULT 1 COMMENT '预估番茄数',
    description TEXT COMMENT '模板描述',
    is_default BOOLEAN DEFAULT FALSE COMMENT '是否为默认模板',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='任务模板表';
```

## 10 部署与配置

- 命令分隔符使用分号（;）而不是 &&

## 11 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| 计时精度不足 | 用户信任度下降 | 使用系统高精度定时器，定期校准 |
| 数据丢失 | 用户统计数据清零 | 提供 JSON 导出/导入功能，自动备份 |
| 跨平台兼容性 | 某些平台无法运行 | 使用 PyQt5 跨平台组件，分平台测试 |
| 用户习惯断裂 | 不再使用应用 | 简化操作流程、连续打卡激励机制 |
| 开发时间不足 | 功能缺失 | 严格按迭代开发，V0.1 即可使用 |
| MySQL 连接问题 | 应用无法启动 | 提供详细的数据库设置指南，连接测试功能 |
| 数据库损坏 | 用户数据丢失 | 自动备份机制，数据恢复功能 |
| 用户隐私担忧 | 用户不信任 | 明确数据本地存储，不上传云端 |
| 竞品功能丰富 | 用户流失 | 专注核心体验，快速迭代 |
| 开发进度延期 | 发布延迟 | 敏捷开发，MVP优先，分阶段交付 |
